# Games Cloud Save

`Games Cloud Save` 是一个基于 `PyQt5` 的多游戏本地存档同步工具。

它的目标是把“不同游戏、不同电脑上的本地存档”统一备份到 GitHub，并提供简单明确的上传、下载流程。

## 当前版本特点

- 支持多游戏独立配置
- 每个游戏独立设置本地存档目录
- 每个游戏独立设置云端备份路径
- 下载覆盖前可先备份本地存档
- 支持覆盖后确认或回退
- 使用“最近一次上传记录”作为对比

## 运行环境

- Windows
- 建议 Python 3.11 及以上
- 依赖见 `requirements.txt`

安装依赖：

```bash
pip install -r requirements.txt
```

启动方式：

```bash
python main.py
```

## 首次使用

1. 填写 GitHub Token
2. 填写仓库名，格式为 `用户名/仓库名`
3. 填写分支名，通常是 `main`
4. 新增一个游戏
5. 选择该游戏的本地存档文件夹
6. 为该游戏填写独立的云端路径

示例：

- `save_sync/Games/save_backup_latest.zip`

## 配置文件

程序运行后会在当前目录生成：

- `games_cloud_save_config.json`

这个文件包含：

- GitHub 仓库信息
- 本地游戏路径
- 设备名
- 最近上传记录
- GitHub Token

## 使用建议

- 在设置页为每个游戏填写独立的云端存档目录名，程序会自动在该目录下使用 `save_backup_latest.zip`
- 不同游戏不要共用同一个云端存档目录

- 使用启动器前，在设置页点击“记录目标窗口”，并在 5 秒内切换到需要监控的游戏或模拟器主窗口
- 启动器始终运行所选 EXE，但只根据记录的窗口特征监控目标窗口关闭，不等待后台残留进程退出
- 目标窗口未立即出现时，启动器会持续等待且不设超时；可在等待窗口中点击“取消监控”退出
- 主程序创建的游戏专用启动器会使用绑定的游戏配置执行云存档下载、启动、窗口监控和上传流程
- 当前游戏配置完成后，可在设置页创建绑定该游戏存档启动器的 Windows 快捷方式

- 设置页输入框失去焦点时会自动保存；也可点击“查看配置”打开程序实际使用的 JSON

- 上传前确认当前选择的是正确的存档文件夹
