# Games Cloud Save

`Games Cloud Save` 是一个游戏本地存档同步工具。

它的目标是把“不同游戏、不同电脑上的本地存档”统一备份到 GitHub，并提供简单明确的上传、下载流程。

## 当前版本特点

- 支持多游戏独立配置
- 每个游戏独立设置云端备份路径
- 下载覆盖前可先备份本地存档
- 支持覆盖后确认或回退
- 使用“最近一次上传记录”作为对比

## 运行环境

- Windows
- 建议 Python 3.11 及以上
- 依赖见 `requirements.txt`

安装依赖：

```bash
pip install -r requirements.txt
```

启动方式：

```bash
python main.py
```

## 首次使用

1. 在设置页选择云端方式（GitHub 或 Gitee）
2. 填写对应平台的 Token
3. 填写仓库名，格式为 `用户名/仓库名`
4. 填写分支名，通常是 `main`
5. 新增一个游戏
6. 选择该游戏的本地存档文件夹
7. 为该游戏填写独立的云端路径

兼容说明：
- 新版优先使用 `provider_type` + 普通仓库名
- 旧版如果把 Gitee 仓库写成 `gitee:用户名/仓库名`，程序仍然兼容识别

## 配置文件

程序运行后会在指定目录生成：

- `games_cloud_save_config.json`

这个文件包含：

- GitHub 仓库信息
- 本地游戏路径
- 设备名
- 最近上传记录
- GitHub Token

## 使用建议

- 在设置页为每个游戏填写独立的云端存档目录名，程序会自动在该目录下使用 `save_backup_latest.zip`
- 不同游戏不要共用同一个云端存档目录
- 使用启动器前，在设置页点击“选择目标窗口”，并从当前正在运行的窗口列表中选择需要监控的游戏或程序窗口；列表项会以“窗口标题（进程名）”显示
- 启动器始终运行所选 EXE，但只根据记录的窗口特征监控目标窗口关闭，不等待后台残留进程退出
- 目标窗口未立即出现时，启动器会持续等待且不设超时；可在等待窗口中点击“取消监控”退出
- 填写完成设置页和“快捷存档游戏启动器”页中的当前游戏信息后，可创建绑定该游戏的桌面启动器。打开启动器后，程序会先执行启动前同步：需要时下载云端存档，不需要时自动跳过，然后再启动游戏；关闭目标游戏窗口后，若检测到存档发生变化，则自动上传云端存档。
- 设置页输入框失去焦点时会自动保存；也可点击“查看配置”打开程序实际使用的 JSON
- 上传前确认当前选择的是正确的存档文件夹
